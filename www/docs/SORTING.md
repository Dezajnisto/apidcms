# Сортировка данных на динамических страницах

> Относится к типам страниц: blog, catalog, landing, price (и любые кастомные)

## Как работает сортировка

При создании или редактировании элемента навигации (страницы) есть поля:

| Поле | Назначение | По умолчанию |
|------|-----------|-------------|
| `sort_field` | Колонка в таблице, по которой сортировать | `created_at` |
| `sort_order` | Направление сортировки (ASC / DESC) | `DESC` |

Значения по умолчанию подходят для блогов и новостей (сначала свежие).
Для страниц типа «прайс» или «каталог» логичнее сортировать по `id ASC`.

## Как настроить

Через админку: **Таблицы → navigation → найти нужную страницу → Редактировать**.

Внизу формы найти поля **sort_field** и **sort_order**:

- `sort_field` = `id` — сортировка по порядку добавления записей
- `sort_order` = `ASC` — по возрастанию (сначала старые)

## Если данные перемешались

Проверь:
1. Какое значение в `sort_field` и `sort_order` у нужной страницы в navigation
2. Есть ли в твоей таблице колонка `sort_order` — если да, она приоритетнее

## Для разработчика

Поведение по умолчанию задаётся в `NavigationItem::getPageConfig()`:

```php
'sort' => [
    'field' => $this->sort_field ?: 'created_at',
    'order' => $this->sort_order ?: 'DESC'
]
```

Чтобы изменить сортировку для конкретной страницы — просто укажи свои `sort_field` и `sort_order` в форме редактирования navigation.

## Лимит записей на странице (items_per_page)

По умолчанию динамические страницы показывают **10 записей**. Если в твоей таблице больше записей, чем 10, остальные не отобразятся.

Где настроить: **Таблицы → navigation → найти страницу → Редактировать → поле items_per_page**.

| Значение | Результат |
|----------|-----------|
| 0 | Покажет все записи (без лимита) |
| 10 | Дефолт, первые 10 записей |
| 20 | Покажет 20 записей |
| 50 | Покажет 50 записей |

Для страниц типа «прайс» или «лендинг» рекомендуется поставить `0` (без лимита) или заведомо большее число.

**Важно:** `items_per_page` влияет только на типы страниц blog, catalog, landing и кастомные типы. Для типа `page` (статическая страница) не используется.
