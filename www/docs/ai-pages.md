# AI-страницы (тип `ai`)

Тип страницы `ai` превращает страницу сайта в чат-интерфейс с искусственным интеллектом. Посетитель задаёт вопрос — ИИ отвечает, используя реальные данные из базы сайта.

## Как это работает

1. Посетитель открывает ai-страницу и видит окно чата
2. Пишет вопрос (например: «Какие стихи есть на сайте?»)
3. ИИ получает контекст — структуру всех таблиц БД, навигацию сайта и содержимое
4. Формирует ответ на основе реальных данных
5. Показывает быстрые ссылки для продолжения диалога

## Создание ai-страницы

1. В админке откройте **Навигация** → **Добавить**
2. Заполните:
   - **Название** — «AI Помощник» (или любое)
   - **URL** — `ai-chat` (или любой другой)
   - **Тип страницы** — выберите `ai`
   - **Статус** — `active`
3. Нажмите **Сохранить**
4. Страница готова: `https://вашсайт.ru/ai-chat`

> **Примечание:** ai-страницы не используют `source_table` и `page_config` — контекст собирается автоматически из всей базы данных.

## Настройки

Все настройки находятся в админке: **Настройки системы** → секция **«🤖 AI Фронтенд (чат для посетителей)»**.

### Галочка «Использовать системные настройки AI»

| Положение | Что происходит |
|---|---|
| ✅ Включена (по умолчанию) | Используется `ai_api_key` и `ai_model` из админки |
| ❌ Снята | Появляются поля для отдельного ключа и модели фронтенда |

### Отдельные настройки (когда галочка снята)

| Поле | Назначение |
|---|---|
| **API ключ фронтенда** | Отдельный API-ключ для ai-страниц |
| **Модель фронтенда** | Модель для ответов посетителям (например `deepseek-chat`) |

### Системный промт

Определяет поведение ИИ. Можно использовать плейсхолдеры:
- `{site_title}` — подставится название сайта
- `{context}` — подставится JSON с данными БД

> ⚠️ Если заполнено поле **«Личность ассистента»** (ниже) — системный промт игнорируется.

**Пример:**
```
Ты — AI-ассистент сайта {site_title}.

{context}

Правила:
1. Отвечай на русском, дружелюбно
2. Используй ТОЛЬКО информацию из контекста
```

## Быстрые ссылки (quick_links)

ИИ может добавлять в ответ блок `[quick_links:...]` с JSON-массивом ссылок. Они отображаются как кликабельные кнопки под сообщением:

```json
[quick_links:[{"label":"Каталог стихов","url":"/poetry"},{"label":"Контакты","url":"/contact"}]]
```

Блок автоматически удаляется из текста ответа и показывается как кнопки.

## Личность ассистента (рекомендуется)

Поле **«Личность ассистента»** в админке — самый мощный инструмент настройки. В отличие от системного промта, контекст БД добавляется **автоматически** — вам не нужно включать `{context}`.

**Что можно задать:**
- 🎭 **Роль и характер** — «Ты — Ирина, менеджер сайта», «Ты — Клавдия, поэтесса»
- 🚫 **Ограничения тем** — «Отвечай только о товарах», «На вопросы не по теме: извинись и откажись»
- 📏 **Длину ответов** — «Будь краткой, не более 3 предложений»
- 💰 **Лидогенерацию** — «При первой возможности предложи оформить заказ»
- 📞 **Контакты** — «В конце диалога дай телефон и форму связи»

**Пример для сайта стихов:**
```
Ты — Клавдия, поэтесса и хозяйка сайта «Стихи на заказ».

Ты говоришь как творческий человек, используешь метафоры.

Правила:
1. Отвечай ТОЛЬКО о поэзии, стихах и творчестве
2. На вопросы не по теме отвечай: «Я здесь только про стихи, дорогой! 😊»
3. Будь эмоциональной, но краткой — не более 3 предложений
4. В конце обязательно предложи: «Хотите заказать стихи? Напишите нам!»
```

**Пример для интернет-магазина:**
```
Ты — Анна, консультант интернет-магазина {site_title}.

Правила:
1. Отвечай только о товарах, ценах, доставке и услугах магазина
2. На вопросы не по теме: «Извините, я здесь помогаю с выбором товаров 😊»
3. Отвечай кратко, по делу
4. После ответа предложи: «Могу помочь оформить заказ?»
5. Если клиент спрашивает о конкретном товаре — опиши его преимущества
6. В конце диалога напомни о бесплатной доставке при заказе от 3000₽
```

> 💡 **Совет:** `{site_title}` в личности подставится автоматически — название сайта из настроек.

> ⚠️ **Важно:** Если поле «Личность ассистента» заполнено, поле «Системный промт» игнорируется. Личность + контекст БД — это всё, что нужно.

## Ограничения

- **Контекст:** в промт включаются все таблицы БД. Для таблиц с ≤100 записей добавляются sample-строки (до 50). Для больших таблиц — только структура.
- **Rate limiting:** не чаще 1 запроса в 2 секунды от одного посетителя (по сессии).
- **API-ключ:** никогда не передаётся клиенту — запросы проксируются через сервер.

## Дизайн

Страница чата использует базовый шаблон сайта (`base.html.twig`) и добавляет:
- Контейнер чата (белая карточка с тенью, max-width 768px)
- Серую цветовую гамму (stone), не конфликтующую с основным дизайном
- Анимированный индикатор печати
- Стилизованные быстрые ссылки (pill-кнопки)

Если нужно изменить дизайн — редактируйте `ai.html.twig` в ядре или в `front/app/views/` проекта (для проектов с кастомным фронтендом).

## Устранение неполадок

| Проблема | Причина | Решение |
|---|---|---|
| «AI не настроен» | Нет API-ключа | Добавить `ai_api_key` в system_settings |
| ИИ выдумывает ответы | Не подставлен контекст | Проверить что в промте есть `{context}` |
| Страница не грузится | Нет шаблона | Скопировать `ai.html.twig` в `front/app/views/` проекта |
| «Слишком частые запросы» | Rate limit | Подождать 2 секунды |
